Search pill · design options

Each option shows three states: no active search/filters · search fields only · search fields + filters.
The pill / badge indicates what is active; the × clears everything at once.

Option A
Dark pill · integrated × inside the pill
When nothing is active the pill is hidden and only the search icon is shown. As soon as a field or filter is set, a dark pill slides in to the left of the icon. The pill contains the summary text and a small circular × button that clears everything.

Cons: Pill and × are small targets on mobile.
Option B
Compound button · icon + label + ×
Idle state: a plain circular icon button. Active state: the button stretches into a pill-shaped compound widget — icon on the left, divider, summary text, then a circular × button on the right — all in a single bordered element.

Option C
Numeric badge on icon + separate clear chip
The search icon carries a small numeric badge (total active count). A separate "2 fields · 3 filters ×" chip appears to the left. The chip is tinted dark and acts as the clear button (or contains an inline ×).

Option D
Individual field/filter tags + "Clear all"
Each active search field and filter is shown as a small individual tag above the icon row. Search tags are neutral; filter tags are tinted blue. Each tag has its own × to remove it individually. A "Clear all" pill appears when there is more than one active item.

Option E
Dark pill · hover toolbox with per-item removal
Same pill as Option A. On hover a small card drops below the pill listing every active search field and filter. Each item shows its field name and value with a circular × to remove it individually. Filters are tinted blue. The pill's own × still clears everything at once. The second and third columns simulate the hover state with the toolbox fully visible.
